- 2026-05-31: On Linux without OpenSSL headers, `make -C benches` can fail because `benches/Makefile` has macOS OpenSSL paths and `-DLIBUS_USE_OPENSSL`. For plaintext issue-167 checks, rebuild `../uWebSockets/uSockets` with `make WITH_LTO=0` (default `LIBUS_NO_SSL`) and compile the harness directly: `clang -O3 -I../uWebSockets/uSockets/src benches/load_test.c ../uWebSockets/uSockets/uSockets.a -lpthread -o benches/load_test`.
